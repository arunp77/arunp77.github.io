1. The Problem Statement

1.1 Supervised Learning Setup

Let the input space be \(\mathcal{X} \subseteq \mathbb{R}^d\) and the output space be \(\mathcal{Y}\) (e.g.\ \(\mathcal{Y}=\mathbb{R}\) for regression, \(\mathcal{Y}=\{0,1\}\) for binary classification). We assume an unknown joint distribution \(P(X,Y)\) over \(\mathcal{X}\times\mathcal{Y}\) from which data are drawn i.i.d.:

$$(x_i, y_i) \overset{\text{i.i.d.}}{\sim} P(X,Y), \quad i=1,\ldots,n$$

Our goal is to find a prediction function \(F:\mathcal{X}\to\mathbb{R}\) that minimises the expected loss (risk):

$$\mathcal{R}[F] = \mathbb{E}_{(X,Y)\sim P}\bigl[L(Y,\,F(X))\bigr]$$

where \(L:\mathcal{Y}\times\mathbb{R}\to\mathbb{R}_{\ge 0}\) is a loss function measuring how wrong prediction \(F(x)\) is compared to true label \(y\).

1.2 Why This Is Hard

The distribution \(P(X,Y)\) is unknown; we only have a finite sample \(\mathcal{D}=\{(x_i,y_i)\}_{i=1}^n\). We cannot compute \(\mathbb{E}[L]\) — only estimate it. Gradient Boosting addresses this by constructing the hypothesis class \(\mathcal{H}\) as an additive ensemble and optimising empirically.

2. Statistical Decision Theory

2.1 The Optimal Predictor

The theoretically optimal predictor minimises the true risk:

$$F^*(x) = \arg\min_{F}\;\mathbb{E}_{Y|X=x}\bigl[L(Y,\,F(x))\bigr]$$

For different loss functions \(F^*\) takes different forms:

📐 Squared Error Loss   \(L(y,\hat y)=(y-\hat y)^2\)

Optimal predictor: \(F^*(x)=\mathbb{E}[Y\mid X=x]\)

Derivation: Minimise \(\mathbb{E}[(Y-c)^2\mid X=x]\) over constant \(c\):

$$\frac{\partial}{\partial c}\mathbb{E}[(Y-c)^2\mid X=x] = -2\mathbb{E}[Y-c\mid X=x] = -2\bigl(\mathbb{E}[Y\mid X=x]-c\bigr)=0$$
$$\Longrightarrow \quad c = \mathbb{E}[Y\mid X=x] \quad \checkmark$$
📏 Absolute Error Loss   \(L(y,\hat y)=|y-\hat y|\)

Optimal predictor: \(F^*(x)=\operatorname{Median}(Y\mid X=x)\)

Derivation: The subgradient condition:

$$\frac{\partial}{\partial c}\mathbb{E}[|Y-c|\mid X=x] = \mathbb{E}[\operatorname{sign}(c-Y)\mid X=x] = 0$$

This equals zero when \(P(Y\le c\mid X=x)=0.5\), i.e.\ \(c\) is the conditional median. \(\checkmark\)

🎯 0-1 Loss (Classification)   \(L(y,\hat y)=\mathbf{1}[y\ne\hat y]\)

Optimal predictor (Bayes classifier):

$$F^*(x)=\arg\max_k\;P(Y=k\mid X=x)$$

2.2 The Approximation Problem

We cannot achieve \(F^*\) because (1) \(P(X,Y)\) is unknown, (2) we restrict \(F\) to a hypothesis class \(\mathcal{H}\), and (3) we have only finite data. Gradient Boosting addresses all three by building \(\mathcal{H}\) as an additive ensemble and optimising empirically.

3. Empirical Risk Minimization

3.1 The Empirical Risk

Since we cannot compute \(\mathcal{R}[F]=\mathbb{E}[L(Y,F(X))]\), we approximate it with training data:

$$\hat{\mathcal{R}}[F] = \frac{1}{n}\sum_{i=1}^{n}L(y_i,\,F(x_i))$$

By the Law of Large Numbers, as \(n\to\infty\): \(\hat{\mathcal{R}}[F]\xrightarrow{a.s.}\mathcal{R}[F]\).

ERM:

$$\hat F = \arg\min_{F\in\mathcal{H}}\;\hat{\mathcal{R}}[F] = \arg\min_{F\in\mathcal{H}}\sum_{i=1}^{n}L(y_i,\,F(x_i))$$

3.2 The Challenge of Function Optimisation

For parametric models (e.g.\ \(F(x;\theta)=\theta^Tx\)) we minimise over \(\theta\in\mathbb{R}^p\) — a finite-dimensional problem solvable by gradient descent. But what if \(F\) cannot be parameterised easily (decision trees, heterogeneous ensembles)? This motivates functional optimisation — optimising directly in the space of functions.

4. From Gradient Descent to Functional Gradient Descent

4.1 Review: Gradient Descent in Parameter Space

Given \(\mathcal{L}(\theta)=\sum_i L(y_i,F(x_i;\theta))\), gradient descent updates:

$$\theta^{(t+1)} = \theta^{(t)} - \eta\,\nabla_\theta\mathcal{L}(\theta^{(t)})$$

The gradient component for parameter \(\theta_j\) is:

$$\frac{\partial\mathcal{L}}{\partial\theta_j} = \sum_{i=1}^n \frac{\partial L(y_i,F_i)}{\partial\theta_j} = \sum_{i=1}^n \frac{\partial L}{\partial F_i}\cdot\frac{\partial F_i}{\partial\theta_j}$$

4.2 The Key Insight: Treat Predictions as Variables

Instead of thinking about parameters, think of the prediction vector:

$$\mathbf{F} = \bigl(F(x_1),\,F(x_2),\,\ldots,\,F(x_n)\bigr)\in\mathbb{R}^n$$

The empirical risk is a function of this vector:

$$\mathcal{L}(\mathbf{F}) = \sum_{i=1}^n L(y_i,\,F_i)$$

The negative gradient at each training point, called the pseudo-residual, is:

$$\boxed{r_i = -\frac{\partial L(y_i,\,F(x_i))}{\partial F(x_i)}}$$

4.3 Functional Gradient Descent

In parameter-space gradient descent we move \(\theta\) in direction \(-\nabla_\theta\mathcal{L}\). In functional space we want to move \(F\) in the direction that most reduces the loss — the negative functional gradient:

$$\nabla_F\mathcal{L}[F]\big|_{x=x_i} = \frac{\partial L(y_i,F(x_i))}{\partial F(x_i)} = -r_i$$

The functional gradient is only defined at training points \(\{x_i\}\). To generalise to new \(x\) values we fit a weak learner \(h_m\) to the pseudo-residuals \(\{(x_i,r_i)\}\). This learner approximates the negative functional gradient everywhere.

💡 Core Connection: Just as gradient descent moves parameters in direction \(-\nabla_\theta\mathcal{L}\), gradient boosting moves the prediction function in direction \(-\nabla_F\mathcal{L}\) by fitting a tree to pseudo-residuals.

4.4 Why Fitting to Negative Gradients Works

For MSE loss \(L(y,F)=\tfrac{1}{2}(y-F)^2\):

$$r_i = -\frac{\partial L}{\partial F_i} = -\frac{\partial}{\partial F}\Bigl[\tfrac{1}{2}(y_i-F)^2\Bigr] = y_i - F(x_i)$$

The pseudo-residual is the ordinary residual — the current prediction error. Fitting the next model to these residuals corrects the current model. For other losses, \(r_i\) is a generalised residual indicating the direction and magnitude of needed adjustment.

Proof: Pseudo-residuals always reduce the loss.
The directional derivative of \(\mathcal{L}\) in direction \(r=(r_1,\ldots,r_n)\):
$$\lim_{\epsilon\to 0}\frac{\mathcal{L}(F+\epsilon r)-\mathcal{L}(F)}{\epsilon} = \sum_{i=1}^n\frac{\partial L(y_i,F_i)}{\partial F_i}\cdot r_i = \sum_{i=1}^n(-r_i)\cdot r_i = -\|r\|^2 \;\le\; 0 \quad\square$$
Moving in direction \(r\) always decreases \(\mathcal{L}\) for small enough \(\epsilon\).

5. The Additive Model

5.1 Forward Stagewise Additive Modeling (FSAM)

Gradient Boosting builds:

$$F_M(x) = F_0(x) + \sum_{m=1}^{M}\nu_m\,h_m(x)$$

where \(F_0\) is the initial model, \(h_m\in\mathcal{H}\) is the \(m\)-th weak learner, \(\nu_m\ge 0\) is a step size, and \(M\) is the number of boosting rounds. Stagewise means at each step \(m\) we fix all previous \(h_1,\ldots,h_{m-1}\) and only optimise for \(h_m,\nu_m\).

5.2 The Initialisation \(F_0\)

The initial model is the best constant prediction:

$$F_0(x) = \arg\min_{c\in\mathbb{R}}\sum_{i=1}^n L(y_i,\,c)$$
LossFormula\(F_0\)Derivation note
MSE \(\tfrac{1}{2}(y-F)^2\) \(\bar y = \tfrac{1}{n}\sum_i y_i\) Set \(\sum(y_i-c)=0\)
MAE \(|y-F|\) \(\operatorname{median}(y_1,\ldots,y_n)\) Median minimises L1
Log-loss \(\log(1+e^F)-yF\) \(\log\dfrac{\bar p}{1-\bar p}\) where \(\bar p=\tfrac{1}{n}\sum y_i\) Set \(\tfrac{e^c}{1+e^c}=\bar p\)
Poisson \(e^F - yF\) \(\log\bar y\) Set \(e^c=\bar y\)

Full derivation for Log-loss initialisation:

$$\sum_{i=1}^n L(y_i,c) = -\sum_i\bigl[y_i c - \log(1+e^c)\bigr]$$
$$\frac{\partial}{\partial c} = -\sum_i y_i + n\cdot\frac{e^c}{1+e^c} = 0 \quad\Rightarrow\quad \frac{e^c}{1+e^c}=\bar p \quad\Rightarrow\quad c = \log\frac{\bar p}{1-\bar p} \quad\checkmark$$

6. Gradient Boosting Algorithm — General Form

────────────────────────────────────────────────────────
Algorithm: Gradient Boosting (General Form)
────────────────────────────────────────────────────────
Input: data {(xᵢ,yᵢ)}ⁿᵢ₌₁, loss L, weak learner class 𝓗, rounds M, lr ν

Step 0 — Initialise:
    F₀(x) = argminc Σ L(yᵢ, c)

For m = 1 to M:

    Step 1 — Pseudo-residuals (negative gradient):
        rᵢₘ = −∂L(yᵢ, F(xᵢ))/∂F(xᵢ) |F=Fₘ₋₁, ∀ i

    Step 2 — Fit weak learner to pseudo-residuals:
        hₘ = argminh∈𝓗 Σᵢ (rᵢₘ − h(xᵢ))²
        # Always use squared error here, regardless of original L

    Step 3 — Line search (optimal step size):
        νₘ = argminν>0 Σᵢ L(yᵢ, Fₘ₋₁(xᵢ) + ν·hₘ(xᵢ))

    Step 4 — Update:
        Fₘ(x) = Fₘ₋₁(x) + νₘ · hₘ(x)

Output: F_M(x)
⚠️ Why always MSE in Step 2? Pseudo-residuals \(r_{im}\) are real-valued "direction signals" that have already encoded the choice of \(L\). We want the weak learner to point in that direction — squared error is the natural measure of how well \(h_m(x_i)\) matches \(r_{im}\).

6.1 Fixed Shrinkage

In practice we often use a fixed learning rate \(\nu\in(0,1]\) instead of an exact line search:

$$F_m(x) = F_{m-1}(x) + \nu\cdot h_m(x)$$

Smaller \(\nu\) means each tree contributes less, requiring more trees \(M\) but giving better generalisation. Empirically \(\nu\in[0.01,\,0.1]\) with large \(M\) (controlled by early stopping) gives optimal results.

6.2 Geometric Interpretation

At each step the algorithm:

  1. Computes the direction of steepest descent in function space: \(-g_i=r_{im}\)
  2. Projects this direction onto hypothesis class \(\mathcal{H}\) by fitting \(h_m\)
  3. Moves the ensemble one step in that direction
$$F_m = F_{m-1} + \nu_m\;\underbrace{\operatorname{proj}_{\mathcal{H}}\bigl[-\nabla_F\mathcal{L}\bigr]}_{h_m}$$

7. Loss Functions — Full Derivations

7.1 Squared Error (L2) — Regression
$$L(y,F)=\tfrac{1}{2}(y-F)^2$$

Gradient:

$$g_i=\frac{\partial L}{\partial F_i}=-(y_i-F_i)=F_i-y_i$$

Pseudo-residual:

$$r_i = -g_i = y_i - F_i \quad\text{(ordinary residual)}$$

Properties: Differentiable everywhere ✓ | Sensitive to outliers (quadratic) ✗ | Optimal \(F^*=\mathbb{E}[Y|X]\) ✓

7.2 Absolute Error (L1) — Robust Regression
$$L(y,F)=|y-F|$$

Subgradient:

$$g_i = \operatorname{sign}(F_i-y_i) = \begin{cases}+1 & F_i>y_i \\ -1 & F_i<y_i \\ 0 & F_i=y_i\end{cases}$$

Pseudo-residual:

$$r_i = \operatorname{sign}(y_i-F_i)$$

Pseudo-residuals are \(\pm 1\) — only the sign of the error matters, not the magnitude. This makes gradient boosting with L1 loss very robust to outliers.

7.3 Huber Loss — Balanced Robustness
$$L_\delta(y,F)=\begin{cases}\tfrac{1}{2}(y-F)^2 & |y-F|\le\delta \\ \delta\bigl(|y-F|-\tfrac{\delta}{2}\bigr) & |y-F|>\delta\end{cases}$$

Gradient:

$$g_i=\begin{cases}F_i-y_i & |y_i-F_i|\le\delta \\ \delta\cdot\operatorname{sign}(F_i-y_i) & |y_i-F_i|>\delta\end{cases}$$

Quadratic (efficient) for small errors, linear (robust) for large errors. \(\delta\) controls the transition — errors within \(\delta\) are treated as L2; beyond \(\delta\) as L1.

7.4 Binary Cross-Entropy (Log-Loss) — Binary Classification

Model outputs log-odds \(F(x)\in\mathbb{R}\); probability via sigmoid:

$$p=\sigma(F)=\frac{1}{1+e^{-F}},\qquad F=\log\frac{p}{1-p}$$
$$L(y,F)=\log(1+e^F)-yF$$

Full gradient derivation:

$$\frac{\partial L}{\partial F} = \frac{\partial}{\partial F}\bigl[\log(1+e^F)-yF\bigr] = \frac{e^F}{1+e^F}-y = \sigma(F)-y = p-y$$
$$\boxed{g_i = p_i - y_i, \qquad r_i = y_i - p_i}$$

The pseudo-residual is the difference between true label and predicted probability. Verify at extremes: if \(y_i=1,\,p_i=0.99\) then \(r_i=0.01\) (tiny correction); if \(y_i=1,\,p_i=0.01\) then \(r_i=0.99\) (large correction). ✓

7.5 Exponential Loss — AdaBoost Connection
$$L(y,F)=\exp(-y\cdot F),\quad y\in\{-1,+1\}$$
$$g_i=-y_i\exp(-y_iF_i), \qquad r_i=y_i\exp(-y_iF_i)$$

When used with decision stumps, gradient boosting with exponential loss is exactly AdaBoost. The sample weight in AdaBoost is \(w_i\propto\exp(-y_iF_{m-1}(x_i))=|r_i|\) — AdaBoost's reweighting is the gradient boosting negative gradient!

7.6 Softmax Deviance — Multi-class Classification

For \(K\) classes, model \(K\) functions \(F_1(x),\ldots,F_K(x)\):

$$p_k(x)=\frac{\exp(F_k(x))}{\sum_{k'}\exp(F_{k'}(x))}$$
$$L(y,F_1,\ldots,F_K)=-\log p_y =-F_y+\log\sum_{k=1}^K\exp(F_k)$$

Gradient w.r.t. \(F_k\):

$$g_{ik}=\frac{\partial L}{\partial F_k}=p_{ik}-\mathbf{1}[y_i=k]$$
$$r_{ik}=\mathbf{1}[y_i=k]-p_{ik}$$

At each boosting round, fit \(K\) separate trees — one per class residual vector.

7.7 Poisson Deviance — Count Data

Model \(\mu=e^F\) ensures positivity:

$$L(y,F)=e^F - y\cdot F$$
$$g_i=e^{F_i}-y_i=\mu_i-y_i, \qquad r_i=y_i-\mu_i=y_i-e^{F_i}$$
7.8 Quantile (Pinball) Loss — Quantile Regression

To predict the \(\tau\)-th quantile (\(\tau\in(0,1)\)):

$$L_\tau(y,F)=\begin{cases}\tau(y-F) & y\ge F \\ (1-\tau)(F-y) & y<F\end{cases}$$
$$r_i=\begin{cases}\tau & y_i\ge F_i \\ -(1-\tau) & y_i<F_i\end{cases}$$

For \(\tau=0.5\): \(r_i=0.5\cdot\operatorname{sign}(y_i-F_i)\) — recovers L1/median regression. Running separate models with different \(\tau\) gives the full conditional distribution of \(Y|X\).

8. Weak Learners — What Can \(h_m\) Be?

Gradient boosting is agnostic to the choice of weak learner. Requirements: (1) \(h_m:\mathcal{X}\to\mathbb{R}\); (2) fitted by minimising squared error to pseudo-residuals. Any regression model qualifies.

🌳 Regression Trees (Most Common)

A tree with \(J\) leaves partitions \(\mathcal{X}\) into regions \(R_1,\ldots,R_J\):

$$h_m(x)=\sum_{j=1}^J c_{mj}\cdot\mathbf{1}[x\in R_{mj}]$$

Leaf value (MSE): \(c_{mj}=\tfrac{1}{|R_{mj}|}\sum_{x_i\in R_{mj}}r_{im}\)

📈 Linear Models

If \(\mathcal{H}=\{x\mapsto\theta^Tx\}\):

$$h_m(x)=\hat\theta_m^T x,\quad \hat\theta_m=\arg\min_\theta\sum_i(r_{im}-\theta^Tx_i)^2$$

The ensemble becomes gradient descent on a linear model — showing GBM generalises GD.

🔮 Kernel Ridge Regression
$$h_m(x)=\sum_{i=1}^n\alpha_i k(x,x_i),\quad \alpha=(K+\lambda I)^{-1}r_m$$

Gradient boosting in a reproducing kernel Hilbert space (RKHS).

🧠 Small Neural Networks

One-hidden-layer networks as weak learners. Each round fits a new small network to residuals. The ensemble can express richer functions than any single network with the same total parameter count.

⚠️ Why "Weak"? A learner that perfectly fits \(\{(x_i,r_{im})\}\) memorises noise. A weak learner (shallow tree) captures the main signal without overfitting. The ensemble of many weak learners achieves low bias; variance is controlled by learner capacity and shrinkage \(\nu\).

9. The Line Search and Optimal Step Size

9.1 Exact Line Search

After fitting \(h_m\), the optimal step size is:

$$\nu_m = \arg\min_{\nu>0}\sum_{i=1}^n L\bigl(y_i,\,F_{m-1}(x_i)+\nu\cdot h_m(x_i)\bigr)$$

This is a 1-dimensional optimisation — much simpler than the original problem.

9.2 Closed-Form for MSE

$$\mathcal{L}(\nu)=\sum_{i=1}^n\tfrac{1}{2}(y_i-F_{m-1}(x_i)-\nu h_m(x_i))^2$$
$$\frac{\partial\mathcal{L}}{\partial\nu} =-\sum_{i=1}^n(y_i-F_{m-1}(x_i)-\nu h_m(x_i))\,h_m(x_i)=0$$
$$\sum_{i=1}^n r_{im}\,h_m(x_i) = \nu\sum_{i=1}^n h_m(x_i)^2$$
$$\boxed{\nu_m = \frac{\sum_{i=1}^n r_{im}\,h_m(x_i)}{\sum_{i=1}^n h_m(x_i)^2}}$$

This is the OLS projection of residuals \(r_{im}\) onto direction \(h_m(x_i)\).

9.3 Per-Leaf Line Search (Tree Base Learners)

For tree-based weak learners we can do a separate line search for each leaf region \(R_{mj}\):

$$\nu_{mj}=\arg\min_{\nu>0}\sum_{x_i\in R_{mj}}L(y_i,\,F_{m-1}(x_i)+\nu)$$
$$F_m(x)=F_{m-1}(x)+\sum_{j=1}^J\nu_{mj}\cdot\mathbf{1}[x\in R_{mj}]$$

9.4 Newton Step Per Leaf

A one-step Newton approximation to the per-leaf 1D optimisation:

$$\nu_{mj}\approx-\frac{f'(0)}{f''(0)} = -\frac{\displaystyle\sum_{x_i\in R_j}g_i}{\displaystyle\sum_{x_i\in R_j}h_i} = \frac{\displaystyle\sum_{x_i\in R_j}r_i}{\displaystyle\sum_{x_i\in R_j}h_i}$$

where \(g_i=\partial L/\partial F_i\) and \(h_i=\partial^2 L/\partial F_i^2\). For MSE: \(h_i=1\), so \(\nu_{mj}=\bar r_j\) — the mean residual in the leaf. This Newton step is the foundation of XGBoost and LightGBM.

10. Regularization in Gradient Boosting

10.1 Shrinkage (Learning Rate)

$$F_m(x) = F_{m-1}(x) + \nu\cdot h_m(x), \quad \nu\in(0,1]$$

Each weak learner contributes only \(\nu\) times its prediction. The model uses more trees to achieve the same training fit, but each tree is less committed — reducing variance.

$$\operatorname{Var}(F_M)\propto \nu^2\cdot M\cdot\operatorname{Var}(h_m)$$

Keeping \(\nu\cdot M\) constant while decreasing \(\nu\) (and increasing \(M\)) reduces variance while maintaining bias.

10.2 Subsampling

At each round, fit \(h_m\) using a random subsample of fraction \(\eta\in(0,1]\) (without replacement):

$$h_m=\arg\min_{h}\sum_{i\in\mathcal{S}_m}(r_{im}-h(x_i))^2, \quad |\mathcal{S}_m|=\lfloor\eta n\rfloor$$

Introduces randomness → reduces correlation between trees. Optimal \(\eta\in[0.5,0.8]\).

10.3 Tree-Specific Regularization

For tree-based weak learners:

TechniqueEffect
Max depth \(D\)Limits tree complexity. Typical \(D\in[3,8]\).
Min leaf size \(n_{min}\)Prevents fitting to small noisy groups.
Min split gain \(\gamma\)Only split if Gain \(>\gamma\). Equivalent to \(\gamma\) per leaf penalty.
L2 on leaf weights \(\lambda\)Penalises large leaf predictions: \(\Omega(h)=\gamma J+\tfrac{\lambda}{2}\sum_j w_j^2\)

With L2 regularization, the optimal leaf weight becomes:

$$\boxed{w_j^* = -\frac{G_j}{H_j+\lambda}}$$

10.4 Early Stopping

for m = 1 to M_max:
    F_m = F_{m-1} + ν·h_m
    val_loss_m = Σ L(y_i, F_m(x_i)) over validation set
    if val_loss has not improved for k rounds:
        return F_{m−k} # best model so far

Combined with small \(\nu\) and large \(M_{max}\), early stopping automatically selects the right model complexity for the given dataset.

10.5 Feature Subsampling

At each round, use only a random subset of features \(\mathcal{F}_m\subset\{1,\ldots,d\}\) with \(|\mathcal{F}_m|=\lfloor p\cdot d\rfloor\). Reduces correlation between trees (each uses different features). Typical \(p\in[0.5,0.8]\).

10.6 Bias-Variance Tradeoff Summary

To reduce Overfitting (high variance)To reduce Underfitting (high bias)
Decrease num_leaves / max_depthIncrease num_leaves / max_depth
Increase min_data_in_leafDecrease min_data_in_leaf

11. Second-Order Gradient Boosting — Newton Boosting

11.1 Motivation

Standard gradient boosting uses only the first derivative \(g_i\). Newton's method uses both first and second derivatives for faster, more accurate optimisation:

$$F \leftarrow F - \frac{g_i}{h_i}$$

The Hessian \(h_i\) (curvature) tells us how quickly the loss changes — higher curvature means smaller, safer steps.

11.2 Taylor Approximation of the Loss

Around \(\hat y_i=F_{m-1}(x_i)\), expand the loss to second order with \(\delta=h_m(x_i)\):

$$L(y_i,\hat y_i+\delta)\approx L(y_i,\hat y_i)+g_i\delta+\tfrac{1}{2}h_i\delta^2$$

Dropping constants, the total objective for the new tree:

$$\tilde{\mathcal{L}}^{(m)} =\sum_{i=1}^n\Bigl[g_i h_m(x_i)+\tfrac{1}{2}h_i h_m(x_i)^2\Bigr]+\Omega(h_m)$$

where \(\Omega(h)=\gamma J+\tfrac{\lambda}{2}\sum_{j=1}^J w_j^2\) is the regularisation term.

11.3 Optimal Leaf Weights — Full Derivation

Group by leaves — let \(I_j=\{i:q(i)=j\}\):

$$\tilde{\mathcal{L}}^{(m)} =\sum_{j=1}^J\Bigl[\underbrace{\textstyle\sum_{i\in I_j}g_i}_{G_j}w_j +\tfrac{1}{2}\underbrace{\Bigl(\textstyle\sum_{i\in I_j}h_i+\lambda\Bigr)}_{H_j+\lambda}w_j^2\Bigr]+\gamma J$$

Define: \(G_j=\sum_{i\in I_j}g_i\) (gradient sum) and \(H_j=\sum_{i\in I_j}h_i\) (Hessian sum).

The objective is separable and quadratic in each \(w_j\). Minimise:

$$\frac{\partial\tilde{\mathcal{L}}}{\partial w_j}=G_j+(H_j+\lambda)w_j=0$$
$$\boxed{w_j^*=-\frac{G_j}{H_j+\lambda}}$$

This is the Newton step: gradient \(G_j\) divided by curvature \(H_j\), with \(\lambda\) stabilising the denominator.

11.4 Tree Quality Score

Substituting \(w_j^*\) back into the objective:

$$\tilde{\mathcal{L}}^* =\sum_{j=1}^J\Bigl[G_j\cdot\Bigl(-\tfrac{G_j}{H_j+\lambda}\Bigr) +\tfrac{1}{2}(H_j+\lambda)\cdot\tfrac{G_j^2}{(H_j+\lambda)^2}\Bigr]+\gamma J$$
$$\boxed{\tilde{\mathcal{L}}^*=-\frac{1}{2}\sum_{j=1}^J\frac{G_j^2}{H_j+\lambda}+\gamma J}$$

Lower (more negative) = better tree structure.

11.5 Split Gain Formula

For a candidate split dividing leaf \((G,H)\) into left \((G_L,H_L)\) and right \((G_R,H_R)\):

$$\text{Before split (1 leaf):}\quad \mathcal{L}^*_{\text{before}}=-\tfrac{1}{2}\frac{G^2}{H+\lambda}+\gamma$$
$$\text{After split (2 leaves):}\quad \mathcal{L}^*_{\text{after}}=-\tfrac{1}{2}\frac{G_L^2}{H_L+\lambda} -\tfrac{1}{2}\frac{G_R^2}{H_R+\lambda}+2\gamma$$
$$\boxed{\text{Gain} =\frac{1}{2}\left[\frac{G_L^2}{H_L+\lambda}+\frac{G_R^2}{H_R+\lambda} -\frac{G^2}{H+\lambda}\right]-\gamma}$$

A split is made only if Gain \(>0\) (improvement exceeds the \(\gamma\) regularisation penalty).

11.6 Why the Hessian Matters — Intuition

Consider two leaves both with gradient sum \(G_j=10\):

Leaf A: \(H_j=100\) (many samples, high curvature)
\(w_A^*=-10/(100+0)\approx-0.10\) → small, cautious update
Leaf B: \(H_j=1\) (few samples, low curvature)
\(w_B^*=-10/(1+0)=-10\) → large, confident update

The Hessian correctly gives Leaf A a smaller update (conflicting gradient signals from many samples) and Leaf B a larger update (confident signal). Without the Hessian, both leaves would get the same weight — a clear mistake.

12. Stochastic Gradient Boosting

12.1 The Algorithm with Subsampling

At each round draw \(\mathcal{S}_m\subset\{1,\ldots,n\}\) of size \(\lfloor\eta n\rfloor\):

$$h_m=\arg\min_h\frac{1}{|\mathcal{S}_m|}\sum_{i\in\mathcal{S}_m}(r_{im}-h(x_i))^2$$

12.2 Unbiasedness

The stochastic functional gradient estimate is unbiased:

$$\mathbb{E}_{\mathcal{S}_m}[r_{im}^{stoch}]=r_{im}^{full}$$

The variance introduced by subsampling acts as regularization — it prevents overfitting to any particular subset of training points.

Friedman showed stochastic gradient boosting with \(\eta\approx 0.5\) often outperforms deterministic gradient boosting — it benefits from both sequential correction and the variance reduction from subsampling.

13. Convergence and Bias-Variance Analysis

13.1 Error Decomposition

$$\mathbb{E}\bigl[(Y-\hat F_M(X))^2\bigr] = \underbrace{\sigma^2}_{\text{irreducible}} + \underbrace{\operatorname{Bias}^2(\hat F_M)}_{\text{approx. + estimation}} + \underbrace{\operatorname{Var}(\hat F_M)}_{\text{estimation variance}}$$

13.2 Training Loss Decreases Monotonically

$$\sum_{i=1}^n L(y_i,F_M(x_i))\;\le\;\sum_{i=1}^n L(y_i,F_m(x_i))\quad\text{for }M>m$$

Each new tree is chosen to reduce the loss — training loss cannot increase.

13.3 Variance via Shrinkage

$$\operatorname{Var}(F_M)\approx \nu^2 M\cdot\operatorname{Var}(h)$$

With subsampling: \(\operatorname{Cov}(h_m,h_{m'})\approx 0\), so \(\operatorname{Var}(F_M)\approx\nu^2 M\cdot\operatorname{Var}(h)=\nu\cdot C\cdot\operatorname{Var}(h)\), which decreases as \(\nu\to 0\).

13.4 Convergence Rates

Under mild conditions (convex loss, rich enough \(\mathcal{H}\)):

$$\mathcal{L}(F_M)-\mathcal{L}(F^*)=O\!\left(\frac{1}{M}\right)\quad\text{(sublinear)}$$

For strongly convex losses with strong convexity constant \(\mu\) and Lipschitz constant \(L_{lip}\):

$$\mathcal{L}(F_M)-\mathcal{L}(F^*) \le\left(1-\frac{\mu}{L_{lip}}\right)^{\!M}\!\bigl(\mathcal{L}(F_0)-\mathcal{L}(F^*)\bigr) \quad\text{(linear / geometric decay)}$$

15.2 Gradient Boosting ↔ AdaBoost (Formal Proof)

AdaBoost uses sample weights \(w_i^{(m)}\) and classifiers \(h_m:\mathcal{X}\to\{-1,+1\}\):

$$w_i^{(1)}=\frac{1}{n},\quad w_i^{(m+1)}=\frac{w_i^{(m)}e^{-\alpha_m y_i h_m(x_i)}}{Z_m}$$

With exponential loss \(L(y,F)=e^{-yF}\):

$$r_i=-g_i=y_i e^{-y_i F_{m-1}(x_i)}=y_i w_i^{(m)}n$$

Pseudo-residuals are proportional to \(y_i w_i^{(m)}\) — exactly AdaBoost's weighted labels. The optimal step size \(\alpha_m\) matches the gradient boosting line search for exponential loss. ∴ AdaBoost = Gradient Boosting with exponential loss. ∎

16. Master Equation Summary

16.1 The Complete Algorithm

GRADIENT BOOSTING — COMPLETE FORM
══════════════════════════════════════════════════════════
Input: {(xᵢ,yᵢ)}ⁿᵢ₌₁, L, 𝓗, M, ν, λ, γ

Initialise:
  F₀(x) = argmin_c Σ L(yᵢ, c)

For m = 1 … M:
  ① Statistics
    gᵢ = ∂L(yᵢ,F̂ᵢ)/∂F̂ᵢ |_{F̂=Fₘ₋₁} (1st derivative)
    hᵢ = ∂²L(yᵢ,F̂ᵢ)/∂F̂ᵢ² |_{F̂=Fₘ₋₁} (2nd derivative)
    rᵢ = −gᵢ (pseudo-residual)

  ② Fit weak learner
    hₘ = argmin_{h∈𝓗} Σ (rᵢ − h(xᵢ))²

  ③ Line search (or fixed ν)
    νₘ = argmin_ν Σ L(yᵢ, Fₘ₋₁(xᵢ) + ν·hₘ(xᵢ))

  ④ [Newton] Leaf weights
    wⱼ* = −Gⱼ / (Hⱼ + λ) where Gⱼ=Σᵢ∈ⱼ gᵢ, Hⱼ=Σᵢ∈ⱼ hᵢ

  ⑤ Update
    Fₘ(x) = Fₘ₋₁(x) + νₘ · hₘ(x)

Output: F_M(x)
